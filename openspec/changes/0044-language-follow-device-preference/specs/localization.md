# Spec Delta: 多语言本地化

## 修改需求：语言选择优先级

### 用户故事
作为用户，我希望 App 跟随手机当前语言显示界面，而不是被地区码强制覆盖。

### 验收规则
1. 系统必须优先根据设备当前语言偏好选择本地化资源，而不是地区码。
2. 设备语言为 `ja`、`ko`、`en`、`zh-Hans/zh` 时，App 必须切到对应的 shipped 语言资源。
3. 地区码可继续用于日志和埋点，但不得决定 UI 语言。
4. 当设备语言不在支持范围内时，系统必须回退到应用支持语言中的最佳匹配；若仍无法匹配，则回退到英文。
5. 语言选择逻辑必须在主要 UI 创建前安装。

### 边界条件
- 当设备语言列表包含地区后缀时，例如 `ja-CN`、`en-JP`，系统必须按语言前缀识别为 `ja`、`en`。
- 当设备设置了 App 级别语言覆盖时，系统应优先遵循该语言偏好。
- 当匹配语言对应的 lproj 缺失时，系统必须安全回退，不得崩溃。
