# Proposal: Pro 付费模式重构

## 背景 / 动机

- 当前付费能力只覆盖去水印，产品表达和实际限制点不一致。
- 拼接调整、涂抹/打码、PDF 导出已经是高价值功能，但没有统一纳入 Pro 权益。
- 去水印弹窗中的价格文案写死为 18 元，无法随 App Store 实际定价和地区货币变化。

## 目标（Goals）

- G1: 将付费能力统一升级为 Pro 版本，并覆盖拼接调整、去水印、涂抹/打码、PDF 导出。
- G2: 免费用户仍可体验部分编辑流程，但在结果生效或导出节点进行 Pro 校验。
- G3: 所有购买弹窗统一展示 Pro 权益列表，并使用 App Store 返回的本地化价格。

## 非目标（Non-goals）

- NG1: 不引入新的订阅模型、账号体系或服务端鉴权。
- NG2: 不重做现有编辑器交互或购买页视觉设计。
- NG3: 不修改现有一次性内购商品 ID。

## 范围（Scope）

- **会改的部分**：
  - 文件/模块：主页付费入口、保存流程、拼接调整完成流程、涂抹/打码保存限制、支付商品信息加载、本地化文案。
  - 行为变化：免费用户可进入拼接调整和涂抹/打码，但在应用结果或导出时弹出 Pro 权益提示；PDF 导出改为 Pro 专属；去水印弹窗改为完整 Pro 权益说明。
- **不会改的部分**：
  - 图片拼接算法和涂抹/马赛克算法本身。
  - ReplayKit 录制、分片收集和恢复购买流程本身。

## 需求概要

- 主界面中的“拼接调整”“涂抹/打码”入口需要显示 Pro 标识。
- 免费用户可进入拼接调整，但点击完成时只有 Pro 用户才能真正应用调整结果。
- 免费用户可使用涂抹/马赛克，但保存时若结果图包含这些效果，必须提示升级 Pro 或移除效果。
- 保存菜单中的 PDF 项需要标注“Pro可用”，未购买时点击后提示升级 Pro。
- 点击“去水印”时，需要展示完整 Pro 权益列表，而不是只介绍去水印。
- 购买弹窗价格必须改为使用 App Store 商品信息中的本地化价格；加载失败时使用无价格兜底文案。

## 验收标准（Acceptance Criteria）

- AC1: 未购买用户进入拼接调整后，点击完成会弹出 Pro 权益弹窗；取消后不应用调整结果。
- AC2: 未购买用户完成涂抹或马赛克后，点击保存图片会弹出 Pro 权益弹窗，并可选择移除效果后继续保存。
- AC3: 未购买用户点击 PDF 导出项时会弹出 Pro 权益弹窗；购买成功后可继续导出。
- AC4: 去水印入口弹窗必须展示完整 Pro 权益列表，并显示 App Store 本地化价格或兜底文案。
- AC5: 已购买用户不受上述限制，当前预览、保存图片和 PDF 导出都不带水印。

## 风险与回滚

- **风险**：Pro 校验点分散，若改造不集中可能出现部分路径漏拦截或购买后续流不一致。
- **回滚方案**：移除统一 Pro 校验层和相关入口标识，恢复当前仅去水印付费的逻辑，同时保留商品信息加载能力但不在 UI 使用。
