# Design: Pro 付费模式重构

## 现状（As-is）

- 主页的付费入口只服务于“去水印”，相关弹窗写死价格文案。
- `PurchaseManager` 只负责请求购买，不对外暴露商品信息和本地化价格。
- 主页中的保存、PDF、去水印和拼接调整流程分别散落在不同方法里，没有统一的 Pro 功能校验入口。
- 涂抹/马赛克结果只写回当前长图，没有独立的“受限效果已使用”决策层。

## 方案（To-be）

- **总体思路**：
  - 保持现有一次性内购商品和本地已购状态不变。
  - 新增一个轻量的 Pro 功能协调层，用统一场景枚举驱动权益文案、价格文案、购买动作和购买后的续流。
  - 将拼接调整、涂抹/打码、PDF 导出、去水印统一接入该协调层。

- **模块与职责**：
  - `PurchaseManager`
    - 继续负责发起购买、恢复购买。
    - 新增商品信息拉取和本地化价格缓存能力。
  - `ProFeature` / `ProAccessCoordinator`
    - 定义各 Pro 能力场景。
    - 生成统一权益列表、弹窗标题和按钮文案。
    - 在需要时触发购买并在成功后续流。
  - `ViewController`
    - 承接主页入口标识、保存菜单、PDF 导出和去水印入口。
    - 在拼接调整应用、保存图片、PDF 导出三个关键时机调用统一 Pro 校验。
  - `EditViewController`
    - 调整完成回传协议，支持由上层决定是否真正应用结果。
  - `MarkupEntryViewController`
    - 为涂抹和马赛克入口增加 Pro 标识展示。
    - `ProPaywallViewController`
      - 以自定义卡片式弹窗页替代简陋的系统 Alert。
      - 负责展示场景说明、权益列表、价格说明和操作按钮。

## 数据流

1. 主页加载后，请求商品信息并缓存价格文案。
2. 用户点击去水印、完成拼接调整、点击保存图片或点击 PDF 导出时，主页调用统一 Pro 校验。
3. 若已购买，直接执行目标动作。
4. 若未购买，展示统一权益弹窗：包含完整 Pro 权益、价格文案和场景化主说明。
5. 用户选择升级 Pro 时，协调层调用 `PurchaseManager.requestPurchase`。
6. 购买成功后，协调层执行传入的续流闭包，继续当前动作。
7. 对于保存图片场景，若用户选择“移除效果”，则回退到未应用涂抹/马赛克的长图再继续保存。
8. 所有 Pro 标识统一使用橙色视觉，主页按钮角标采用更小的悬浮标签，二级列表入口采用左上角斜角标，避免遮挡文字和箭头布局。

## 接口/数据结构变更

- 新增 `enum ProFeatureGate`，至少包含：
  - `removeWatermark`
  - `applyStitchAdjustment`
  - `saveMarkupEffects`
  - `exportPDF`
- 新增商品信息查询接口，返回：
  - 当前价格字符串
  - 是否已成功加载商品信息
  - 失败时的兜底文案
- `EditViewController.onConfirm` 改为支持返回 `currentRanges` 给上层并由上层决定是否真正应用。
- `ViewController` 新增状态字段区分：
  - 当前展示图是否包含涂抹/马赛克效果
  - 未应用受限效果时的基础长图，用于“移除效果”回退

## 关键取舍

- 免费用户仍可进入编辑器，而不是在入口直接阻断，以满足产品要求中的“先体验后拦截”。
- “移除对应效果”首版按移除全部涂抹/马赛克结果实现，不区分单独移除涂抹或马赛克，避免引入复杂分层状态。
- 价格只使用 App Store 商品信息，不接额外云配置，避免增加新的依赖面。

## 兼容性与迁移

- 现有 `purchase_status` Keychain 数据保持不变，已购用户自动视为 Pro。
- 不需要迁移分片数据或编辑结果数据。

## 可观测性

- 复用现有 `AppLogger`，新增商品信息加载成功/失败、各 Pro 场景弹窗触发、购买成功续流的日志。

## 回滚策略

- 若线上出现误拦截或续流异常，可回退 `ViewController` 的统一 Pro 校验调用和新弹窗逻辑，恢复当前仅去水印付费路径。
