# Specs: long-screenshot

## 变更主题：统一 Pro 权益与动态价格

### 场景

当前应用只将“去水印”作为付费能力，实际高价值编辑和导出能力没有统一纳入 Pro。产品需要把拼接调整、去水印、涂抹/打码、PDF 导出统一归为 Pro 版本权益，并将所有相关弹窗升级为统一的 Pro 权益说明。

### 要求

- 系统必须将以下能力统一视为 Pro 权益：拼接调整结果应用、去水印、涂抹/打码效果保存、PDF 导出。
- 主界面中的“拼接调整”“涂抹/打码”入口必须带有 Pro 标识。
- 所有 Pro 标识统一使用橙色视觉；主页按钮角标不得遮挡按钮文字，二级入口列表中的 Pro 标识应以左上角斜角标形式展示，右侧箭头仍保持垂直居中。
- 保存菜单中的 PDF 导出项必须标注“Pro可用”。
- 免费用户点击“拼接调整”入口时可以进入编辑页；但点击完成时，只有 Pro 用户才能真正应用新的拼接裁剪结果。
- 免费用户可以使用涂抹和马赛克编辑器；但如果当前结果图包含这些效果，则保存图片前必须提示升级 Pro 或移除对应效果。
- 免费用户点击 PDF 导出项时，系统必须提示升级 Pro 或取消。
- 点击“去水印”按钮时，系统必须展示完整 Pro 权益列表，而不是只说明单一去水印能力。
- 主界面的去水印入口应明确使用 Pro 语义和橙色视觉，体现其属于 Pro 权益而非普通操作按钮。
- 当购买成功或恢复购买成功后，系统必须立即同步首页 Pro 状态，无需重启应用；至少应刷新去水印入口显隐和当前结果图的水印展示。
- Pro 权益提示不得继续使用简陋的系统提示文案堆叠方式，而应展示更完整的自定义弹窗页，至少包含标题、当前场景说明、权益列表、价格说明和操作按钮。
- Pro 弹窗页必须明确表达这是永久权益：一次支付即可长期使用，不是订阅，也不会自动续费。
- 所有购买相关弹窗都必须优先展示 App Store 返回的本地化价格；当价格暂时不可用时，系统必须回退到不含金额的通用升级文案。

### 验收

- 未购买用户在拼接调整完成、涂抹/马赛克后保存、点击 PDF 导出三条路径上都能看到统一风格的 Pro 权益弹窗。
- 已购买用户在上述路径上都不会再被拦截，且结果图预览与导出都不带水印。
- 设置页恢复购买成功后，返回首页或保持首页在前台时，首页去水印入口和结果图预览都能立即反映最新 Pro 状态。
- 价格文案随 App Store 商品信息变化，不再依赖代码中的固定金额字符串。
