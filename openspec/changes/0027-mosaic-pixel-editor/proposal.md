# Proposal: 马赛克页接入真实像素化编辑

## 背景 / 动机

- 当前 MosaicViewController 仍然只是占位页，只能展示原图和说明文案，无法完成真实打码。
- 用户已经明确需要在长图上滑动应用像素化效果，并支持调节粗细、透明度和撤销回退。
- 现有 MarkupViewController 已经解决了长图编辑的输入、输出和缩放链路，本次更适合复用该交互模型，而不是另起一套图片回写流程。

## 目标（Goals）

- G1: MosaicViewController 接收长图并输出编辑后的长图，回写链路与 MarkupViewController 保持一致。
- G2: 用户单指在图上滑动时，局部区域实时呈现像素化马赛克效果。
- G3: 页面提供粗细、透明度调节，以及撤销/重做能力。

## 非目标（Non-goals）

- NG1: 本次不引入工程化的图层编辑历史持久化。
- NG2: 本次不支持橡皮擦、选区、局部移动或再次进入后继续编辑。
- NG3: 本次不改动主页的水印、保存、PDF 导出和拼接算法。

## 范围（Scope）

- **会改的部分**：
  - 文件/模块：MosaicViewController、MarkupEntryViewController、OpenSpec 文档。
  - 行为变化：马赛克页从占位骨架升级为真实像素化编辑页；入口文案不再标注“开发中”。
- **不会改的部分**：
  - PencilKit 涂抹实现、拼接编辑页、主页导出流程。

## 需求概要

- 马赛克页应继续接收一张长图作为输入，并在完成后输出一张新的长图。
- 用户单指滑动时，应在对应区域实时应用像素化效果。
- 页面应提供粗细和透明度两个可见调节控件。
- 参数控件应以易理解的百分比显示当前滑杆进度，而不是直接暴露 pt 数值。
- 页面应支持撤销和重做最近的马赛克笔划。
- 页面应继续支持长图浏览和局部放大精修。

## 验收标准（Acceptance Criteria）

- AC1: 从编辑方式中间页进入马赛克页后，可以看到长图并进行局部缩放浏览。
- AC2: 用户单指在图片上滑动时，对应区域会实时出现像素化马赛克效果。
- AC3: 调整粗细和透明度后，新生成的笔划应按照当前参数生效。
- AC4: 点击撤销和重做后，页面预览与最终导出结果都应与历史操作一致。
- AC5: 点击完成后，应回传编辑后的长图并关闭整个编辑流程。

## 风险与回滚

- **风险**：
  - 长图较长时，实时绘制马赛克预览可能带来性能压力。
  - 若预览坐标与导出坐标换算不一致，可能导致显示位置和最终导出位置偏移。
- **回滚方案**（如何恢复到变更前状态）：
  - 恢复 MosaicViewController 的占位实现，并把主规格中的马赛克要求退回骨架描述。
