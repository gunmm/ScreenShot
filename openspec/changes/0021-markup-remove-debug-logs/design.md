# Design: 移除 Markup 编辑页调试日志

## 现状（As-is）

- MarkupViewController 在布局居中、背景滚动层同步、缩放和滚动回调中调用统一的 logGeometry 方法。
- logGeometry 会格式化多组几何参数，并通过 AppLogger 输出 [Markup] 前缀调试日志。
- 类中还保留了仅用于限流日志输出的 scrollLogCounter 状态。

## 方案（To-be）

- 总体思路：
  - 删除 logGeometry 及其格式化辅助方法。
  - 删除所有日志调用点。
  - 删除仅为日志存在的 scrollLogCounter。
- 模块与职责：
  - MarkupViewController 继续负责画布布局、缩放同步和导出。
  - AppLogger 不做改动，因为本次只清理单页开发期日志。
- 数据流：
  1. 画布缩放或滚动时，仍执行现有居中与背景同步。
  2. 不再进入日志分支，也不再拼装几何字符串。

## 接口/数据结构变更

- 无对外接口变更。
- 仅删除内部私有方法和私有状态。

## 关键算法/阈值

- 无算法变化。
- 不修改最小/最大缩放、偏移同步和导出映射规则。

## 兼容性与迁移

- 无数据迁移。
- 仅影响运行期日志输出内容。

## 可观测性

- 本次变更目标就是移除该页调试日志，不新增替代日志。
- 通过代码搜索 [Markup] 字符串和 Swift 诊断检查确认清理完整。

## 回滚策略

- 若后续需要恢复排查日志，可从本次变更中恢复 logGeometry 相关实现。
