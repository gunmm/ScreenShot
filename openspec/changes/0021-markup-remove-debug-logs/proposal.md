# Proposal: 移除 Markup 编辑页调试日志

## 背景 / 动机

- 当前 Markup 编辑页在缩放、滚动和布局同步过程中会输出带有 [Markup] 前缀的几何调试日志。
- 这些日志属于开发期排查信息，继续保留会增加运行期日志噪音，且对最终用户行为没有价值。

## 目标（Goals）

- G1: 删除 Markup 编辑页中所有 [Markup] 调试日志输出。
- G2: 删除仅为这些日志服务的调用逻辑和遗留状态，且不改变编辑器交互行为。

## 非目标（Non-goals）

- NG1: 不调整 Markup 编辑页的缩放、滚动、导出或 UI 行为。
- NG2: 不删除其他模块的日志。

## 范围（Scope）

- 会改的部分：
  - 文件/模块：MarkupViewController 中的日志方法、调用点及相关辅助状态。
  - 行为变化：运行时不再输出 [Markup] 几何调试日志。
- 不会改的部分：
  - 涂鸦绘制与导出逻辑。
  - 其他页面的日志系统与 AppLogger 实现。

## 需求概要

- Markup 编辑页不应再输出 [Markup] 调试日志。
- 仅用于生成这些日志的内部辅助方法应一并移除。
- 滚动、缩放、双击放大和导出行为应保持不变。

## 验收标准（Acceptance Criteria）

- AC1: MarkupViewController 中不存在 [Markup] 日志字符串。
- AC2: 不再保留仅为 [Markup] 日志服务的私有方法、计数器或调用语句。
- AC3: 不引入新的 Swift 编译错误。

## 风险与回滚

- 风险：若误删了与布局同步有关的语句，可能影响底图与画布对齐。
- 回滚方案：恢复本次变更中删除的日志辅助方法与调用点。
