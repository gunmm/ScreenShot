# Tasks: 保存成功后的自动日志上传

> 规则：任务必须可执行、可验证、可按顺序完成。完成一项打勾一项。

## 实施清单

- [x] 更新规格：新增本次变更的 proposal/design/tasks/spec 文档
- [x] 为 CloudKit 自动上传增加 24 小时限流和日志快照上传能力
- [x] 在保存成功回调接入静默自动上传入口
- [x] 更新主规格中的自动上传要求
- [x] 完成文件级错误检查与关键手工验证
- [ ] 归档（移动到 `openspec/archive/`）

## 自测步骤（手工）

- [ ] Step 1: 首次安装或清除自动上传时间后，保存一张长截图，确认不会影响成功提示，且 CloudKit 收到一条 `AutoLogUpload` 记录。
- [ ] Step 2: 在 24 小时内再次保存长截图，确认不会新增第二条 `AutoLogUpload` 记录。
- [ ] Step 3: 将本地上次上传时间改为超过 24 小时前，再次保存，确认会重新上传。
- [ ] Step 4: 断网或 CloudKit 失败时保存长截图，确认用户仍只看到原有保存成功提示，App 本地日志记录上传失败。
