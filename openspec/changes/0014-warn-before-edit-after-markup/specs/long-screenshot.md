# Specs: long-screenshot

## 新增需求：涂抹后的拼接调整确认

### 场景
用户先对当前长图做了涂抹/打码，然后再次点击“拼接调整”。由于拼接调整会基于原始分片重新生成结果图，已涂抹效果不会自动保留。

### 要求

- 系统必须识别当前结果图是否已经应用过涂抹修改。
- 当用户点击“拼接调整”时：
  - 若当前结果图未做过涂抹，系统应直接进入拼接调整页。
  - 若当前结果图已做过涂抹，系统必须先展示确认提示。
- 确认提示必须明确说明继续拼接调整会丢弃已涂抹效果。
- 用户取消提示后，系统必须保留当前已涂抹结果图，不得发生跳转或重算。
- 用户确认继续后，系统应按既有拼接调整流程进入编辑页。

### 验收

- 可以通过“拼接完成后直接进入编辑”和“涂抹后再进入编辑”两条路径观察到不同表现。
- 警告只应在当前结果图已做过涂抹时出现。
