# Proposal: 初赛 Demo 最小闭环（可演示、可惊艳、可讲清）

## 背景 / 动机

比赛初赛要求在 `2026-03-03` 前交付可演示 Demo，且评分维度强调：

- 痛点洞察力（清晰用户与刚需场景）
- 产品创新性（差异化与亮点）
- Demo 完成度（主流程顺畅 + 生成效果惊艳）
- 商业潜力（增长空间与商业逻辑）

当前项目的核心能力（跨 App 录制 → 有效帧筛选 → OpenCV 重叠裁剪拼接 → 输出长图）已经具备，但整体仍偏“技术 Demo”，需要补齐产品化闭环与叙事材料。

## 目标（Goals）

- **G1（主流程顺滑）**：用户能在 3 分钟内完成一次稳定演示：开始录制 → 去目标 App 滚动 → 回来一键生成 → 一键分享/保存。
- **G2（效果更惊艳）**：生成结果更“可分享”（至少提供：分享导出；更“干净”的裁边/编辑能力放到后续变更）。
- **G3（可讲清差异化）**：在 Demo 中明确对比 iOS 自带整页截图的覆盖范围限制，突出“跨 App + 智能拼接”的卖点。
- **G4（材料可交付）**：形成可复用的演示脚本与 PPT 大纲，满足辅助材料要求。

## 非目标（Non-goals）

- **NG1**：不做云端/账号/上传分享（初赛阶段先坚持本地隐私友好）
- **NG2**：不在本次引入大规模 AI 模型能力（如 OCR、内容理解、自动摘要）
- **NG3**：不做复杂编辑器（涂抹、马赛克、多图排版等）——可作为复赛方向

## 范围（Scope）

### 会改的部分（产品化最小闭环）

- **引导与状态**：把“怎么录、去哪录、录完怎么回”的信息讲清楚
- **分享导出**：生成长图后支持系统分享面板（比只保存相册更像生产力工具）
- **错误与降级提示**：对“没录到 / 权限 / 拼接失败或降级”的提示更清晰
- **（可选）基础裁边**：提供简单开关或默认策略，去掉顶部/底部固定区域，提升观感
- **演示物料**：演示脚本、PPT 大纲、样张要求

### 不改的部分（保持稳定）

- 采集与拼接的核心算法结构不重做（OpenCV 与 Stitcher 先保守迭代）
- App Group 存储路径不变

## 需求概要（用户可见）

- 用户首次打开 App 即能看懂如何开始一次“跨 App 长截图”的录制与生成。
- 生成完成后可一键分享（保存相册仍保留）。
- 失败时能明确知道下一步怎么做（例如：去系统设置开权限 / 重新录制 / 检查是否下滑）。
- （可选）生成结果更干净，减少顶部/底部重复 UI 对视觉的干扰。

## 验收标准（Acceptance Criteria）

- **AC1（主流程）**：在真机上可稳定演示：录制 10-30 秒滚动内容 → 回到 App → 一键生成 → 出图 → 一键分享成功。
- **AC2（体验）**：生成期间 UI 不假死；有明确的进度/状态；按钮状态合理（不可重复触发）。
- **AC3（惊艳点）**：至少具备一个“用户肉眼可见”的加分点：分享导出 / 结果历史（编辑能力可后续补齐）。
- **AC4（材料）**：`openspec/changes/0002-contest-demo/specs/` 内存在演示脚本与 PPT 大纲文档。

## 风险与回滚

- **风险**：新增 UI/状态逻辑引入回归（按钮不可用、状态错乱）
- **回滚方案**：
  - 分享导出可视为增量功能，不影响旧流程；可通过隐藏入口回滚
  - 视觉增强（如裁边/编辑）若引入，需默认可关闭并可回滚
