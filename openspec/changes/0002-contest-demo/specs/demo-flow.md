# Demo Flow（初赛最小闭环）

## 目标用户与场景（用于痛点评分）

### 目标用户

- 海外内容生产/运营人员（跨境电商运营、客服、社区运营、内容创作者）

### 典型场景（选 1 个作为主叙事）

- **场景 S1：客服对话/订单沟通留证**：把跨页聊天/工单滚动内容一键整理成可分享长图
- **场景 S2：订单/物流信息留存**：滚动的订单详情/物流节点做成一张图方便团队协作
- **场景 S3：攻略/评论流分享**：把多屏攻略/评论拼成一张便于分享

## 用户流程（用户可见）

### 流程 F1：从录制到分享（主演示流程）

1. 打开 App，看到“如何开始”的引导卡片
2. 点击系统录制入口开始录制（选择本 App 的广播扩展）
3. 切到目标 App，持续向下滚动 10-30 秒
4. 回到本 App，点击“一键生成长截图”
5. 生成成功后：
   - 预览结果
   - 点击“分享”打开系统分享面板（推荐展示：保存到相册/分享给聊天工具）

### 流程 F2：失败与补救（必须可解释）

- 若没有录到任何分片：
  - UI 提示“没有录到内容”，并给出建议（确认已开始录制、保持连续下滑、回到 App 再生成）
- 若拼接失败：
  - UI 提示“拼接失败”，给出建议（避开动态内容、滚动更慢、更连续）
- 若相册权限拒绝：
  - UI 提示“权限被拒绝”，引导去系统设置开启

## 差异化陈述（用于创新性评分）

### 对比 iOS 自带整页截图

- 自带整页截图通常只覆盖 Safari/部分场景，且对跨 App 内容无能为力
- 本产品核心差异化：
  - **跨 App 捕获**
  - **智能筛帧（只保留有效下滑帧）**
  - **重叠检测裁剪拼接（减少重复）**

## 验收用例（Acceptance Tests）

### AT-1 主流程顺畅

- 前置：真机安装、相册权限允许（或可在过程中授权）
- 操作：按流程 F1 完成一次录制与生成
- 期望：
  - 生成不假死
  - 出图可预览
  - 分享面板可打开并可选择至少一种分享渠道

### AT-2 未录到 chunk 的可解释性

- 前置：不进行录制或录制时间极短
- 操作：点击生成
- 期望：
  - 明确提示“未找到分片/未录制”
  - 给出下一步建议

### AT-3 拼接失败的可解释性

- 前置：录制动态内容（视频流、强动画），导致匹配容易失败
- 操作：点击生成
- 期望：
  - 明确提示“拼接失败或已降级，可能出现重复”
  - 可跳转到“预览分片”用于调试
