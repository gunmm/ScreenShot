# Proposal: 编辑放大镜清晰度修正

## 背景 / 动机

- 当前编辑页已经有拖动期放大镜，但用户反馈放大结果模糊且变形，无法用于精细判断接缝。
- 现有实现直接截取较宽的横向区域，再塞进固定比例的放大镜容器，导致内容被压缩和拉伸。

## 目标（Goals）

- G1: 放大镜中的接缝区域保持正确长宽比，不再发生变形。
- G2: 放大镜聚焦接缝附近的小范围区域，提升拖动时的清晰度和可判读性。

## 非目标（Non-goals）

- NG1: 不新增全局缩放或双指手势。
- NG2: 不改动编辑页的 ranges 计算逻辑。

## 范围（Scope）

- 会改的部分：
  - 文件/模块：EditViewController 中放大镜采样与渲染逻辑。
  - 行为变化：放大镜改为等比例采样、更窄区域高倍显示。
- 不会改的部分：
  - 主页面拼接流程。
  - ImageStitcher 拼接算法。

## 需求概要

- 放大镜采样区域必须与放大镜容器保持一致的宽高比。
- 放大镜应围绕当前拖动接缝中心点采样，而不是截取整行宽度。
- 放大镜显示时不得拉伸原图内容。
- 调整后拖动体验应比现有版本更清晰，便于看清接壤处。

## 验收标准（Acceptance Criteria）

- AC1: 放大镜中的图像不再出现明显横向或纵向拉伸。
- AC2: 接缝附近文字或边线在放大镜中比当前版本更清晰可辨。
- AC3: 顶部和底部手柄拖动路径都沿用同一套清晰显示逻辑。

## 风险与回滚

- 风险：若采样窗口过窄，用户可能一次看不到足够上下文。
- 回滚方案：恢复到上一版放大镜采样窗口逻辑。
