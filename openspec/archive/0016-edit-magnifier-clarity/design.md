# Design: 编辑放大镜清晰度修正

## 现状（As-is）

- 当前放大镜通过截取 superview 中接缝附近的一个矩形区域生成快照。
- 该矩形横向几乎覆盖整个分片宽度，但放大镜视图本身是固定的 2:1 比例。
- 由于采样区域与展示区域比例不一致，UIImageView 使用 scaleToFill 时会出现拉伸变形；同时采样范围过大，也会摊薄局部细节。

## 方案（To-be）

- 总体思路：
  - 将采样区域改为围绕接缝中心点的固定比例窗口。
  - 采样窗口的宽高比与放大镜容器一致，再按固定倍率放大显示。
- 模块与职责：
  - ChunkContainer：计算接缝中心点、生成等比例采样区域、更新放大镜位置。
  - SeamMagnifierView：以不拉伸原图的方式承载采样结果。
- 数据流：
  1. 拖动手柄产生当前接缝中心点。
  2. 按放大镜容器比例反推出较小采样窗口大小。
  3. 对采样窗口做边界裁切后生成快照。
  4. 将快照按原比例显示到放大镜中。

## 接口/数据结构变更

- 仅调整 ChunkContainer 内部常量与私有方法。
- 不修改外部协议和 ViewController 调用方式。

## 关键算法/阈值

- 使用固定 zoomScale，将 magnifierSize 反推为更小的 sampleSize。
- sampleRect 以 seamPoint 为中心，超出左右边界时自动夹紧。
- UIImageView 不再使用会导致变形的填充策略。

## 兼容性与迁移

- 无数据迁移需求。
- 仅影响拖动时视觉反馈。

## 可观测性

- 继续以手工拖动观察为主，不新增日志。

## 回滚策略

- 如效果不佳，回退采样窗口和 contentMode 改动即可。
