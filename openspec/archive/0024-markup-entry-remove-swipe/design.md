# Design: 移除编辑方式中间页滑动删改

## 现状（As-is）

- MarkupEntryViewController 已经使用 UITableView 展示编辑入口。
- 当前列表项支持 leading swipe 编辑和 trailing swipe 删除。
- 同时存在编辑 alert 和空状态提示，用于配合运行时删除场景。

## 方案（To-be）

- **总体思路**：
  - 保留列表化的数据结构和 UITableView 展示。
  - 删除滑动相关代理方法，以及与其配套的编辑弹窗和空状态逻辑。
  - 页面职责回到“展示入口并导航到对应编辑器”。

- **模块与职责**：
  - MarkupEntryViewController：只维护入口列表和点击跳转，不再承担入口管理。

## 数据流

1. 页面加载时初始化默认入口数组。
2. table view 渲染列表。
3. 点击某一项时，根据目标类型 push 对应控制器。
4. 列表项滑动时不再暴露任何操作。

## 接口/数据结构变更

- 保留 Entry 数据结构。
- 删除滑动编辑/删除所需的方法和视图状态。

## 关键算法/阈值

- 无新增算法。

## 兼容性与迁移

- 不涉及数据迁移。

## 可观测性

- 不新增日志。

## 回滚策略

- 恢复 swipe actions 和编辑弹窗逻辑。
