# Spec Delta: 多语言本地化

## 新增需求：按国家选择语言

### 用户故事
作为美国、日本、韩国、中国的用户，我希望应用在我的国家环境下默认展示对应语言，而不是继续显示中文或不完整翻译。

### 验收规则
1. 系统必须按地区码选择语言：
   - `US -> en`
   - `JP -> ja`
   - `KR -> ko`
   - `CN -> zh-Hans`
2. 当地区码不在上述映射内时，系统必须回退到应用已支持语言中的最佳匹配；若无法匹配，则回退到英文。
3. `NSLocalizedString` 读取到的核心用户可见文案必须走上述语言选择结果。
4. shipped 语言资源至少包含 `en.lproj`、`zh-Hans.lproj`、`ja.lproj`、`ko.lproj`。
5. 设置页、打赏页、恢复购买、购买失败、加载中等核心文案在上述四国环境下不得显示中文硬编码。

### 边界条件
- 地区码存在但对应语言资源缺失时，系统必须安全回退，不得崩溃。
- 语言选择逻辑必须在主要 UI 构建前安装，避免首屏和后续页面语言不一致。
- 非用户可见注释、日志和调试文案不要求全部进入四语言翻译范围。
