# Specs: Markup 撤销按钮初始状态修复

## 场景
用户进入涂抹编辑页时，需要通过导航栏按钮准确感知当前是否存在可撤销或可重做的笔迹操作，避免在空白画布上看到错误的可点击状态。

## 需求
1. 首次打开 MarkupViewController 且未产生任何笔迹时，撤销按钮必须禁用。
2. 首次打开 MarkupViewController 且未产生任何笔迹时，重做按钮必须禁用。
3. 当存在至少一笔可撤销内容时，撤销按钮必须可用。
4. 当存在至少一笔可重做内容时，重做按钮必须可用。
5. 导航栏按钮状态必须与 PencilKit 当前撤销栈能力保持同步，不能依赖页面初始默认值。

## 边界
1. 空白画布包含系统初始化流程时，不应因此把撤销按钮错误置为可用。
2. 用户执行撤销或重做后，按钮状态必须在当前页面内立即刷新。
3. 本次变更不要求新增跨会话恢复、历史记录持久化或自定义撤销栈。
