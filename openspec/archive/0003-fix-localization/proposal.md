# Proposal: 修复中文系统下显示英文的本地化回退问题

## 背景 / 动机

- **为什么要改？** 当前 App 在中文 (zh-Hans) 系统的设备上运行会显示英文。
- **痛点**：由于项目缺乏明确的 `zh-Hans.lproj` 目录，且 Xcode 默认开发语言（Development Region）为 `en`，系统在没找到 `zh-Hans` 时触发了回退机制，降级加载了英文资源，导致国内用户体验受损。

## 目标（Goals）

- G1: 确保在系统语言为简体中文（zh-Hans）的设备上，App 能够正确显示中文本地化文案。
- G2: 遵循标准的 iOS 多语言本地化规范，为 `zh-Hans` 提供独立的 `Localizable.strings` 等资源。

## 非目标（Non-goals）

- NG1: 不修改现成的所有中/英文字符串内容，仅解决语言包加载和回退的配置问题。
- NG2: 不增加对其他语言（如繁体中文、日文等）的新支持。

## 范围（Scope）

- **会改的部分**：
  - 文件系统：新增 `zh-Hans.lproj` 目录，将 `Base.lproj` 中的需要本地化的文件复制到新目录。
  - 工程配置：修改 `LongScreenShot.xcodeproj/project.pbxproj`，将新增的中文资源文件注册到项目中。
- **不会改的部分**：
  - 核心业务代码 (`*.swift`)
  - 英文本地化内容 (`en.lproj`)

## 需求概要

1. 系统语言设置为中文时，App 的 UI 文本呈现中文。
2. 不影响系统语言为英文或其他不包含的语言时的正常（回退到英文）显示。

## 验收标准（Acceptance Criteria）

- AC1: 编译运行 App，当设备或模拟器语言设置为 `简体中文` 时，应用内的“开始录制”、“设置”等文案正确显示为中文。
- AC2: 当设备或模拟器语言设置为 `English` 时，应用正确显示英文。

## 风险与回滚

- **风险**：对 `project.pbxproj` 的直接修改如不慎可能导致工程文件损坏无法打开。
- **回滚方案**：通过 Git 直接丢弃 `project.pbxproj` 的更改，并删除 `zh-Hans.lproj` 目录即可恢复。
