# Proposal: 编辑拖动放大镜预览

## 背景 / 动机

- 当前 EditViewController 支持拖动上下手柄调整每张分片的裁剪范围。
- 但用户在拖动时只能直接看原尺寸拼接视图，接缝位置细节不够明显。
- 当相邻分片内容相似、边界像素差异细微时，用户很难准确判断当前裁剪线是否正好落在最佳衔接处。

## 目标（Goals）

- G1: 用户在拖动顶部或底部手柄时，可以看到当前接缝附近的局部放大预览。
- G2: 放大预览要随着拖动实时更新，并在结束拖动后立即消失，不干扰常规浏览。

## 非目标（Non-goals）

- NG1: 不引入整个编辑页的缩放手势或新的全局预览模式。
- NG2: 不改变现有裁剪区间计算规则和 Done 回传数据结构。

## 范围（Scope）

- 会改的部分：
  - 文件/模块：EditViewController 中的 ChunkContainer 交互层。
  - 行为变化：拖动裁剪手柄期间显示局部放大镜，帮助观察接缝细节。
- 不会改的部分：
  - ImageStitcher 的拼接与裁剪算法。
  - 主页面进入编辑与回传后的重新拼接流程。

## 需求概要

- 仅在用户拖动顶部或底部手柄时显示放大镜。
- 放大镜必须优先展示当前手柄附近的图像细节，并覆盖裁剪边界线位置。
- 放大镜内容要与当前拖动后的裁剪位置同步更新。
- 放大镜应避免遮住手指正下方的接缝区域，优先悬浮在拖动点上方。
- 拖动结束、取消或失败时，放大镜必须立即隐藏。

## 验收标准（Acceptance Criteria）

- AC1: 选中分片并开始拖动顶部或底部手柄时，界面出现局部放大镜预览。
- AC2: 放大镜中的内容会随着手柄移动实时更新，能持续反映当前接缝附近图像。
- AC3: 结束拖动后，放大镜立即消失，原有编辑交互不受影响。
- AC4: 未选中分片、未开始拖动时，不显示放大镜，也不影响现有点击选中逻辑。

## 风险与回滚

- 风险：若放大镜频繁重绘整张大图，可能造成拖动卡顿。
- 回滚方案：移除 ChunkContainer 中新增的放大镜视图和拖动期更新逻辑，恢复当前仅靠原图可视区域调整的行为。
