# 长截图手动编辑规格

## 1. 业务场景
用户完成屏幕录制并生成长截图后，如果发现某一两处接缝对齐得不好（存在内容重复或断层），希望能够通过一个“编辑”选项进入微调模式。
微调模式下，用户能看到构成整张长图的源分片序列，并可以上下拉动各张图的“遮罩/切割边界”，以确定每张图要展示的确切范围。

## 2. 界面与交互流（Flow）
1. 在生成完成后的 `ViewController` 底部，除了 `Share` 和 `Save` 外，增加 `Edit` 按钮。
2. 点击 `Edit` 弹出一个全屏 `EditViewController`：
   - **顶部**：左侧 [取消]，右侧 [确认]
   - **内容区**：垂直滚动视图展示所有的源图，它们按顺序上下排列。
   - **核心交互**：每相邻两张图之间，有一个“调整条”。拖动该条，可以改变上方图的 `Bottom Crop` 或下方图的 `Top Crop`。
   *由于同时控制两张图较为复杂，本版优先实现独立调节每张图的 `Top Offset` 和 `Bottom Offset`。*
3. 调整后实时在视图上反映出来。
4. 点击 [确认] 时，将所有的 `(start, end)` 返回给上一页，并执行真正的 `ImageStitcher` 重绘。

## 3. 验收细化（Acceptance Criteria）
1. **无需多配**：即便有 50 张图，也不能出现严重卡顿（采用 UIImageView 蒙版或者裁剪显示的方法）。
2. **状态维护**：如果用户进入编辑但直接点取消，主页面不应重绘，状态不变。如果点击了确认，系统必须重新 `stitch()`，如果由于极端的切割范围导致整体高度 <= 0，需要提示用户“无效的分段”。
