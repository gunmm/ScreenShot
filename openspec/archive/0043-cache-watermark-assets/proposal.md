# Proposal: 缓存水印文字 tile 与二维码图

## 背景 / 动机

- 当前水印逻辑每次生成预览图或导出图时，都会重新绘制文字 tile 并重新生成二维码图片。
- 用户希望优化这部分逻辑的性能与耗时。

## 目标（Goals）

- G1: 缓存平铺水印使用的文字 tile，避免重复离屏绘制。
- G2: 缓存固定二维码图片，避免重复走 Core Image 生成流程。

## 非目标（Non-goals）

- NG1: 不改变现有水印视觉样式。
- NG2: 不改变 Pro 去水印逻辑。

## 范围（Scope）

- **会改的部分**：
  - 文件/模块：`LongScreenShot/ViewController.swift`
  - 行为变化：内部增加缓存，减少重复生成文字 tile 和二维码图片。
- **不会改的部分**：
  - 导出尺寸
  - 保存/PDF/付费流程

## 需求概要

- 当文字、字号和 scale 未变化时，应复用上次生成的文字 tile。
- 当二维码 URL 和目标尺寸未变化时，应复用上次生成的二维码图片。
- 缓存失效时必须自动重建，不能影响导出正确性。

## 验收标准（Acceptance Criteria）

- AC1: 非 Pro 用户预览与导出行为保持不变。
- AC2: 相同参数下重复生成展示图时，不再重复构建文字 tile。
- AC3: 相同参数下重复生成展示图时，不再重复生成二维码图片。

## 风险与回滚

- **风险**：
  - 缓存 key 设计不完整会导致样式变化后仍复用旧图。
- **回滚方案**（如何恢复到变更前状态）：
  - 回退到每次现算 tile 和二维码图片的实现。
