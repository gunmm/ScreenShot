# Proposal: 编辑放大镜显示真实接缝内容

## 背景 / 动机

- 上一版将放大镜采样改为直接裁切单张原图像素，稳定性提升了，但用户反馈放大镜中的内容不再是预期的接缝区域。
- 接缝本质上位于上下两张分片之间，只裁一张图无法正确反映实际接壤处。

## 目标（Goals）

- G1: 放大镜内容回到真实接缝语义，能够同时反映接缝上下两侧的分片内容。
- G2: 保留上一版的稳定性，不再依赖整个编辑区截图。

## 非目标（Non-goals）

- NG1: 不修改放大镜的 160 方形尺寸和顶部角落固定位置。
- NG2: 不修改拖动裁剪的 ranges 计算逻辑。

## 范围（Scope）

- 会改的部分：
  - 文件/模块：EditViewController 与 ChunkContainer 的放大镜预览生成逻辑。
  - 行为变化：放大镜改为离屏拼接接缝上下两侧内容，而非单张图局部裁切。
- 不会改的部分：
  - 主页面拼接结果生成逻辑。
  - 编辑页手柄交互方式。

## 需求概要

- 放大镜必须展示当前被调整接缝的上下两侧内容。
- 顶部手柄应展示“上一张底部 + 当前张顶部”的组合；底部手柄应展示“当前张底部 + 下一张顶部”的组合。
- 没有相邻分片时，应退化为当前图片接缝附近内容，但仍保持接缝居中语义。
- 放大镜仍需保持稳定、清晰、固定在顶部角落。

## 验收标准（Acceptance Criteria）

- AC1: 拖动顶部手柄时，放大镜能看到当前分片顶部与上一分片底部的真实接缝关系。
- AC2: 拖动底部手柄时，放大镜能看到当前分片底部与下一分片顶部的真实接缝关系。
- AC3: 放大镜内容比上一版更符合用户对“接缝处”的预期，同时不恢复明显抖动。

## 风险与回滚

- 风险：若上下两侧像素窗口选取过大或过小，可能让 seam 对齐感不自然。
- 回滚方案：恢复到上一版单张原图稳定裁切方案。
