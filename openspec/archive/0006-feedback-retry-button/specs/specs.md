# Specs: 反馈提交失败重试按钮

## 用户故事

**US-01**：作为用户，当反馈提交失败时，我希望在错误提示中直接点击"重试"，一步发起重试，而无需手动再次点击页面上的"提交"按钮。

**US-02**：作为用户，当我不想重试时，我希望点击"取消"关闭错误提示并保留我已输入的内容，以便稍后决策。

## 边界条件

- 仅在 CloudKit 上传失败（`success == false`）时展示重试按钮。
- 网络恢复后重试应能正常上传，逻辑与首次提交完全一致。
- 重试期间显示相同的加载 Alert（"正在上传..."）。

## 验收标准

| ID   | 场景               | 预期结果                                   |
|------|--------------------|--------------------------------------------|
| AC1  | 提交失败           | Alert 同时有"重试"和"取消"两个按钮          |
| AC2  | 点击"重试"         | 重新发起上传，期间显示加载中 Alert          |
| AC3  | 重试成功           | 弹出成功 Alert，确认后关闭 FeedbackVC      |
| AC4  | 重试再次失败       | 再次弹出失败 Alert，仍提供"重试"选项       |
| AC5  | 点击"取消"         | 关闭 Alert，停留页面，输入内容保留         |
| AC6  | 正常提交成功流程   | 不受影响                                   |
