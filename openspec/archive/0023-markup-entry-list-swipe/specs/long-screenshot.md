# Specs: long-screenshot

## 新增需求：编辑方式中间页列表化与滑动操作

### 场景

随着编辑入口数量增加，静态按钮式中间页不再适合作为长期结构。系统需要把编辑方式中间页升级为一个从顶部向下展示的列表页，并为列表项提供滑动编辑与删除能力。

### 要求

- 编辑方式中间页必须使用列表形式从顶部向下展示入口项。
- 列表项必须由统一数据源驱动，便于继续增加更多入口。
- 点击列表项后，系统必须继续进入对应编辑页。
- 列表项必须支持滑动操作，并至少提供编辑与删除两个动作。
- 删除所有列表项后，页面必须展示空状态提示。

### 验收

- 中间页会以列表而非静态按钮形式展示当前所有编辑入口。
- 列表项的副标题不应因布局限制被裁切。
- 滑动编辑后，修改结果会立即反馈到列表展示。
- 滑动删除后，列表与空状态展示应保持一致。
