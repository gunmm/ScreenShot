# 长截图端到端规格（包含编辑功能）

## 1. 业务场景
该模块是一个允许用户从系统级的屏幕录制（ReplayKit / Broadcast Upload Extension）中连续捕获屏幕滚动帧，并在录制结束后将所有帧无缝拼接成一张长图的工具方案。

### 新增功能（编辑）
拼接完成后，由于 OpenCV 特征匹配有时会因为滚动速度不均或出现大量相同色彩导致拼接瑕疵，用户可以点击“Edit / 编辑”进入手动裁剪模式，通过滑块分别调整相邻每张切片的上下裁剪位置，达到完美无缝拼接。

### 新增功能（涂鸦 / 打码）
拼接完成后，用户还可以点击“涂鸦 / 打码”先进入一个编辑方式选择页，再从中进入基于 PencilKit 的涂抹编辑界面，或进入一个为后续真实像素化能力预留的马赛克编辑骨架页。涂抹编辑界面支持长图滚动、缩放和精修，完成后会把笔迹按原图分辨率合成为新的长图。

### 编辑顺序提示
若用户已经对当前结果图做过涂鸦 / 打码，再点击“拼接调整”时，系统必须先明确提示：继续拼接调整会基于原始分片重新生成长图，并丢弃当前已涂抹效果；只有在用户确认后才允许进入拼接调整。

## 2. 核心架构与数据流

### 2.1 录制与分片收集
- **`SampleHandler`**: 接收 ReplayKit 发送的视频帧 (`CMSampleBuffer`)。
- **`FrameProcessor`**: 对每一帧进行对比计算，只有出现滚动特征时（与上一帧差异足够大），才将其视作有效的分片（Chunk）。
- **持久化**: 分片以 JPEG 格式存入 App Group 的共享目录中。

### 2.2 自动拼接（Stitching）
由主 App 中的 `ViewController` 触发。
- **读取分片**：从共享目录读出所有图片，按编号排序。
- **计算裁剪区间 (`calculateValidRanges`)**：
  1. 遍历相邻图片对 $(Img_i, Img_{i+1})$。
  2. 交由 `OpenCVWrapper` 使用特征点匹配技术，查找 $Img_{i+1}$ 顶部在 $Img_i$ 底部匹配的位置及位移(`offsetY`)。
  3. 计算得到每张图片要截取的有效范围：`validRanges[(start, end)]`。
- **绘图 (`stitch(withRanges:)`)**：
  1. 根据 `validRanges` 累加出长截屏的总高度。
  2. 创建 `UIGraphicsImageContext`。
  3. 遍历图片，将每张图片在 `validRanges` 确定的部分按顺序通过 `CoreGraphics` 裁剪后画入上下文中。

### 2.3 手动编辑（Edit）
由用户观察拼接结果后主动触发。
- **`EditViewController`** (互动大更)：
  1. 接收 `images: [UIImage]` 和 `initialRanges: [(start: Int, end: Int)]`。
  2. 使用 `UIStackView` 在 `UIScrollView` 内将所有分片连贯堆叠，**模拟最终成品的无缝拼贴**。
  3. 各张片的 `ChunkContainer` 配置上下两根可供拖拽操作的 Handle (横线区)，拦截 `UIPanGestureRecognizer` 手势指令。
  4. 拖拽期间：利用 `currentRanges` 的改变实时作用在内部图片的位置偏移（TopConstraint offset）和视图窗的高（HeightConstraint）限制上，以达成**所见即所得、滑动自动缝合**的互动效果。
  5. 当用户拖动顶部或底部裁剪手柄时，编辑页必须显示一个跟随更新的局部放大镜，帮助用户以清晰且不失真的方式观察接缝附近的细节位置；放大镜应使用 160 x 160 的方形预览并固定在编辑区域左上角或右上角，其内容应优先基于原始图片像素生成真实接缝预览，能够同时反映接缝上下两侧的分片内容，结束拖动后必须立即消失。
    当前放大倍数应在已稳定可用的基础上略高于上一版，以便更精细地观察 seam 细节，同时保留必要上下文。
  6. 点击 Done 回传调整好的 `newRanges`。
- **重新拼接与应用**：
  1. `ViewController` 收到 `newRanges`，后台线程重新调用 `ImageStitcher.stitch(withRanges:)`。
  2. 更新覆盖主界面的展示大图。
  3. 如果当前展示结果图已经应用过涂抹，用户再次进入该流程前必须先经过确认提示。

## 2.4 保存、导出与付费（Pro Paywall）
- 免费基础版：未付费用户使用 App 时无试用期概念。在生成长截图并在主界面预览时，图片会包含全屏重复倾斜的水印。品牌文字继续承担大面积斜向平铺水印；二维码不再跟随每个旋转 tile 重复出现，而应作为单个、正向、直接展示的独立水印入口叠加在图片右下角位置，以换取更稳定的扫码条件。该方案不得追加底部 footer 或改变原始导出尺寸。免费用户可直接保存普通图片，但导出内容仍会保留相同的全屏平铺水印；若二维码生成失败，系统必须自动退化为纯文字水印。
- 水印实现细节：在文字内容、字号和 scale 未变化时，系统应复用最近一次生成的文字 tile；在二维码 URL 和目标尺寸未变化时，系统应复用最近一次生成的二维码图片，以减少重复渲染开销。
- 高级版（Pro）：用户付费解锁后，“去水印”按钮隐藏，图片预览与导出时都不再带有全屏水印。Pro 权益统一包含：去水印、涂抹/打码效果保存、PDF 导出。
- **限制时机**：
  - **拼接调整**：所有用户都可进入 `EditViewController` 调整裁剪范围，并在点击 Done 后直接应用新的拼接结果。
  - **涂抹 / 打码**：免费用户可进入编辑器并完成修改；但如果当前结果图包含这些效果，则在主页点击“保存图片到相册”前必须提示升级 Pro 或移除效果。
  - **PDF 导出**：免费用户点击“分享为 PDF（Pro可用）”时必须提示升级 Pro 或取消。
  - **去水印**：主界面“去水印”按钮必须展示完整 Pro 权益列表，而不是只介绍单一去水印能力。
- **价格来源**：购买弹窗中的价格不得写死，必须优先使用 App Store 商品信息返回的本地化价格；若商品信息暂时不可用，则需回退为不带金额的通用升级文案。
- **导出格式**：
  - **保存为图片**：通过请求相册权限直接写入到系统相册中。
  - **分享为 PDF**：通过 `UIGraphicsPDFRenderer` 绘制同尺寸单页 PDF，存放于临时目录后，通过 `UIActivityViewController` 调起分享面板供用户保存到“文件”应用或分享给其他人。这是由于 iOS 相册不支持直接保存 PDF 文件所采用的方案。
- **评分邀请**：保存图片成功后，用户关闭成功提示时，可按本地状态决定是否展示评分邀请；若用户已评价或已明确拒绝“不再提示”，后续保存成功都不得再次弹出评分邀请。

## 2.5 编辑方式入口与涂鸦编辑（Markup）
- **`MarkupEntryViewController`**:
  1. 作为“涂鸦 / 打码”流程的模态导航根页，由主页点击入口后先行展示。
  2. 页面必须使用从顶部向下展示的列表结构承载编辑入口，而不是静态按钮布局。
  3. 列表项至少包含标题、副标题和进入提示，并由统一数据源驱动，便于后续继续增加更多入口；各入口项之间应有明确分组间距，不连成同一块，且当前列表项高度应保持一致。
  4. “涂抹”“马赛克”入口必须在右侧展示 Pro 标识。
  5. 点击“涂抹”后 push 到 `MarkupViewController`；点击“马赛克”后 push 到 `MosaicViewController`。
  6. 列表页当前只负责展示入口和导航跳转，不承担入口项的编辑或删除管理。
- **`MarkupViewController`**:
  1. 接收 `rawStitchedImage` 作为待编辑原图。
  2. 使用 `PKCanvasView` 承载绘制、滚动与缩放交互。
  3. 将原图的 `UIImageView` 放在编辑视口的底层，并根据 `PKCanvasView` 的缩放比例、滚动偏移和内容边距同步显示位置，确保底图与笔迹共享相同的视觉几何。
  4. 初始状态按可用宽度适配整张长图，并允许用户通过双指缩放进行局部精修。
  5. 底图层与 `PKCanvasView` 必须共享同一组最小/最大缩放系数，布局刷新后也不得出现缩放边界漂移。
  6. 当用户达到最小或最大缩放边界后，继续双指缩放不应出现弹性回弹。
  7. 运行时不应保留该编辑页用于几何排查的 `[Markup]` 开发期调试日志。
  8. 若该页面是从 `MarkupEntryViewController` push 进入，返回行为应回到中间选择页；点击 Done 时仍需关闭整个编辑流程。
  9. 撤销与重做按钮必须与 `PKCanvasView` 当前撤销栈能力保持同步：空白画布首次进入时两者均为禁用，存在可撤销或可重做内容时才可点击。
  10. 点击 Done 时，将显示坐标系中的 `PKDrawing` 等比映射回原图像素坐标系，与原图合成为新的长图。
- **`MosaicViewController`**:
  1. 接收与 `MarkupViewController` 相同的原始长图输入，并在完成后输出新的长图给主页。
  2. 页面必须支持长图滚动、缩放与局部精修，便于在长图任意位置精确打码。
  3. 用户单指在图片上滑动时，对应区域必须实时呈现像素化马赛克效果。
  4. 页面必须提供粗细和透明度调节能力，且新笔划必须按当前参数生效。
  5. 页面必须支持撤销和重做最近的马赛克笔划。
  6. 点击 Done 时，系统必须把马赛克笔划按原图坐标系与像素化图层合成为新的长图，并回传主页。
- **主页应用结果**:
  1. `ViewController` 将合成后的新图写回 `rawStitchedImage`。
  2. 主界面继续沿用既有水印规则进行展示和导出。
  3. 当当前结果图已应用涂抹或马赛克时，主界面需要记住该状态，用于后续进入拼接调整前触发警告和保存前的 Pro 校验。

## 3. 边界条件与容错处理
1. **录片不足**：至少需要 >=2 个分片才能触发拼接，否则提示录制时间过短。
2. **OpenCV 匹配失败**：如果两个图片找不到明显的重叠特征点（置信度过低），会采用 Full Appending （不裁剪直接叠加）作为安全降级方案，以免丢失内联容错数据。用户可通过 Edit 修正。
3. **负高度或负重叠**：如果用户或者 OpenCV 将上一图的底部切在下一图的顶部还要靠上的位置，会通过高度传播算法（Debt Propagation）使得负重叠被前方分片“吸收”，避免在画布上绘制出负坐标的画面。
4. **编辑界面无效输入**：保证 Top Slider 的值不会大于 Bottom Slider。
5. **相册保存安全性**：由于相册授权回调及完成回调并不保证处于主线程，任何写入系统相册的方法（如 `UIImageWriteToSavedPhotosAlbum`）及产生的 UI 变更，必须严格 dispatch 到 `DispatchQueue.main.async`，防止由于线程违规导致“Unknown error”保存失败。
6. **评分邀请去重**：用户一旦在评分邀请中选择“不再提示”或“去给好评”，该决定必须持久化，后续保存成功时不得再次展示评分邀请。
7. **涂鸦编辑精度**：涂鸦编辑器必须保证底图与笔迹在任意支持的缩放级别下保持稳定对齐，且底图层与笔迹层的最小/最大缩放边界及边界反馈必须一致，导出结果不能因滚动层同步误差出现偏移。
8. **涂鸦撤销状态**：涂鸦编辑器首次进入且画布为空时，撤销/重做按钮必须禁用；用户绘制、撤销、重做后，按钮状态必须立即反映当前撤销栈能力。
9. **涂抹后再编辑提醒**：若当前结果图已经做过涂抹，用户再次点击“拼接调整”时必须先收到“继续会丢弃已涂抹效果”的确认提示；取消后必须保留当前结果图不变。
10. **马赛克编辑精度**：马赛克编辑器必须保证显示预览与最终导出的坐标换算一致；用户看到的打码位置、粗细和透明度不应在导出后发生明显偏移。
